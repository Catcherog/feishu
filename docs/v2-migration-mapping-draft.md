## 8. 迁移对账与验证

### 8.1 对账规则

| 对账项 | 方法 |
|--------|------|
| 记录总数 | 源表记录数 = V2 表记录数 + 重复项 + 失败项 |
| 字段完整性 | 必填字段非空率 = 100% |
| 关联完整性 | customer_link、project_link、resource_link 必须指向真实记录 |
| 重复项 | 单独列出，不静默丢弃 |
| 失败项 | 记录到 Data_Quality_Issue 表 |

### 8.2 迁移后旧表处理

| 阶段 | 旧表状态 | 操作 |
|------|---------|------|
| 迁移中 | 只读 | 不修改旧表数据 |
| 迁移验证 | 只读 | 对账完成 |
| 验证通过 | ARCHIVED | 旧表名加前缀 `[ARCHIVED]`，权限设为只读 |
| V2 稳定运行 | ARCHIVED | 保留 6 个月后评估是否删除 |

### 8.3 legacy 字段保留

所有 V2 记录保留：
- `legacy_record_id`：旧表 record_id
- `legacy_source`：旧表名

用于追溯和回滚。
