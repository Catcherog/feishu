### 0.3 命名规范

- **字段键（Field Key）**：英文 `snake_case`，不依赖中文显示名
- **显示名（Display Name）**：中文
- **自动编号格式**：`{前缀}-{YYYYMMDD}-{序号}`（3位补零）
- **关联字段**：以 `_link` 后缀标识，存储为关联记录 ID
- **时间字段**：以 `_at` 后缀标识日期时间，`_date` 后缀标识纯日期

### 0.4 通用审计字段

以下字段为所有业务表的通用审计字段，不在各表中重复展开：

| 字段键 | 显示名 | 类型 | 必填 | 说明 |
|--------|--------|------|------|------|
| `created_at` | 创建时间 | 日期时间 | 是 | 记录创建时间，由系统写入 |
| `updated_at` | 更新时间 | 日期时间 | 是 | 记录最后修改时间，由系统写入 |
| `legacy_record_id` | 旧记录ID | 文本 | 否 | 迁移自旧表时的 record_id，用于对账 |
| `legacy_source` | 旧表来源 | 文本 | 否 | 迁移自旧表的表名 |

> 注：`created_at` / `updated_at` 在飞书中可使用「创建时间」「修改时间」系统字段，但为统一 Schema 管理和 API 可控性，V2 显式创建为日期时间字段。

## 1. Customer 客户表

### 1.1 目的

保存客户的权威身份、需求和关系状态。一个客户可关联多个项目。客户状态描述商业关系阶段，不复制项目具体交付状态。

### 1.2 字段字典

| # | 字段键 | 显示名 | 飞书字段类型 | 必填 | 说明 | 约束 / 选项 |
|---|--------|--------|-------------|------|------|------------|
| 1 | `customer_id` | 客户编号 | 自动编号 | 是 | 稳定业务 ID，不使用 record_id | 格式：`CUST-{YYYYMMDD}-{###}` |
| 2 | `customer_name` | 客户姓名 | 文本 | 是 | 正式姓名，昵称存入备注 | 非空，≤20 字符 |
| 3 | `phone` | 手机号 | 文本 | 是 | 统一格式，中国大陆或国际标准 | 需标准化校验，用于去重匹配 |
| 4 | `wechat_name` | 微信号 | 文本 | 否 | 不作为唯一匹配键 | — |
| 5 | `source_channel` | 来源渠道 | 单选 | 是 | 客户来源 | 枚举：小红书、微信、官网、小程序、转介绍、其他 |
| 6 | `relationship_status` | 客户状态 | 单选 | 是 | 商业关系状态，见客户状态机 | 枚举：新线索、跟进中、已确认需求、待定金、已成交、服务中、已完成、复购/转介绍、已流失 |
| 7 | `intent_level` | 意向等级 | 单选 | 是 | 客户意向强度 | 枚举：高、中、低、未知 |
| 8 | `service_type` | 服务类型 | 多选 | 否 | 期望拍摄类型 | 枚举：个人、双人、品牌、创作、其他 |
| 9 | `preferred_style` | 偏好风格 | 多选 | 否 | 结构化风格标签 | 枚举：新中式、电影感、复古、日系、清新、暗黑、商务、国潮、其他 |
| 10 | `budget_min` | 预算下限 | 数字 | 否 | 预算范围下限 | ≥0，整数，单位：元 |
| 11 | `budget_max` | 预算上限 | 数字 | 否 | 预算范围上限 | ≥budget_min，可为空 |
| 12 | `preferred_date_start` | 期望档期起 | 日期 | 否 | 期望拍摄时间范围起点 | — |
| 13 | `preferred_date_end` | 期望档期止 | 日期 | 否 | 期望拍摄时间范围终点 | ≥preferred_date_start |
| 14 | `owner` | 跟进负责人 | 人员 | 是 | 当前跟进负责人 | 飞书用户 |
| 15 | `last_contact_at` | 最后跟进时间 | 日期时间 | 否 | 由真实跟进动作更新 | — |
| 16 | `next_action_at` | 下次跟进时间 | 日期时间 | 否 | 下一次应跟进时间 | 用于 F1 自动化查询 |
| 17 | `latest_summary` | 最新跟进摘要 | 多行文本 | 否 | AI 可生成，人工确认后写入 | ≤500 字符 |
| 18 | `privacy_level` | 隐私级别 | 单选 | 是 | 控制信息可见性 | 枚举：普通、敏感、高敏 |
| 19 | `created_at` | 创建时间 | 日期时间 | 是 | 审计字段 | 系统写入 |
| 20 | `updated_at` | 更新时间 | 日期时间 | 是 | 审计字段 | 系统写入 |
| 21 | `legacy_record_id` | 旧记录ID | 文本 | 否 | 迁移对账用 | — |
| 22 | `legacy_source` | 旧表来源 | 文本 | 否 | 迁移对账用 | — |

### 1.3 飞书字段配置说明

- **`customer_id`（自动编号）**：
  - 前缀：`CUST-`
  - 日期格式：`{YYYYMMDD}`
  - 序号：3 位补零（001-999）
  - 示例：`CUST-20260715-001`
  
- **`relationship_status`（单选）**：
  - 颜色建议：新线索（灰）、跟进中（蓝）、已确认需求（青）、待定金（黄）、已成交（绿）、服务中（浅绿）、已完成（深绿）、复购/转介绍（紫）、已流失（红）
  - 状态变更触发 S5 自动化

- **`phone`（文本非电话类型）**：
  - 使用文本类型以兼容国际格式（如 +86 <PHONE_EXAMPLE>）
  - 应用层负责标准化和校验

### 1.4 视图规范

| 视图名称 | 视图类型 | 筛选条件 | 排序 | 说明 |
|---------|---------|---------|------|------|
| 全部客户 | 表格 | 无 | created_at 降序 | 默认视图 |
| 今日待跟进 | 表格 | `next_action_at` ≤ 今天 且 `relationship_status` ≠ 已完成 且 ≠ 已流失 | next_action_at 升序 | F1 自动化数据源 |
| 跟进中客户 | 表格 | `relationship_status` = 跟进中 | next_action_at 升序 | 活跃跟进池 |
| 待定金客户 | 表格 | `relationship_status` = 待定金 | updated_at 降序 | 重点转化对象 |
| 服务中客户 | 表格 | `relationship_status` = 服务中 | owner 分组 | 当前服务中 |
| 已流失客户 | 表格 | `relationship_status` = 已流失 | updated_at 降序 | 回收站 |
| 高意向客户 | 看板 | `intent_level` = 高 且 `relationship_status` ∈ {新线索, 跟进中, 已确认需求, 待定金} | — | 按状态分组看板 |
| 按负责人分组 | 看板 | 无 | — | owner 分组 |
| 高敏客户 | 表格 | `privacy_level` = 高敏 | created_at 降序 | 隐私管控 |

## 2. Project 项目表

### 2.1 目的

表示一次正式交付。客户付定金或负责人确认立项后创建。一个项目关联一个客户，可关联多个资源安排和策划案。

### 2.2 字段字典

| # | 字段键 | 显示名 | 飞书字段类型 | 必填 | 说明 | 约束 / 选项 |
|---|--------|--------|-------------|------|------|------------|
| 1 | `project_id` | 项目编号 | 自动编号 | 是 | 稳定业务 ID | 格式：`PRJ-{YYYYMMDD}-{###}` |
| 2 | `customer_link` | 关联客户 | 关联 | 是 | 必须关联一个 Customer 记录 | 关联到 Customer 表，单选 |
| 3 | `project_name` | 项目名称 | 文本 | 是 | 建议自动生成后允许人工修改 | 建议格式：`{客户名}-{类型}-{日期}` |
| 4 | `project_type` | 项目类型 | 单选 | 是 | 拍摄类型 | 枚举：客片、创作、品牌、其他 |
| 5 | `project_status` | 项目状态 | 单选 | 是 | 见项目状态机 | 枚举：草稿、策划中、策划已批准、资源确认中、待拍摄、拍摄完成、后期制作、客户确认、已交付、已归档 |
| 6 | `shoot_date_start` | 拍摄开始时间 | 日期时间 | 否 | 可为空 | — |
| 7 | `shoot_date_end` | 拍摄结束时间 | 日期时间 | 否 | 可为空 | ≥shoot_date_start |
| 8 | `location_text` | 拍摄地点 | 文本 | 否 | 首期允许文本，后续可关联资源 | ≤100 字符 |
| 9 | `project_owner` | 项目负责人 | 人员 | 是 | 项目总负责人 | 飞书用户 |
| 10 | `photographer` | 摄影师 | 人员 | 否 | 团队内人员可用人员字段 | 飞书用户 |
| 11 | `planning_status` | 策划状态 | 单选 | 是 | 策划案进度 | 枚举：未创建、草稿、待审核、已批准、需修改 |
| 12 | `planning_link` | 关联策划案 | 关联 | 否 | 关联 Planning_Document | 关联到 Planning_Document 表，多选（支持多版本） |
| 13 | `delivery_due_at` | 交付截止日 | 日期 | 否 | 交付 SLA | 用于 F2 自动化超期查询 |
| 14 | `risk_level` | 风险等级 | 单选 | 是 | 项目风险 | 枚举：正常、关注、高风险 |
| 15 | `risk_reason` | 风险原因 | 多行文本 | 否 | 风险触发依据 | 当 risk_level ≠ 正常时建议填写 |
| 16 | `folder_url` | 项目文件夹 | URL | 否 | 项目云盘/素材库链接 | — |
| 17 | `created_at` | 创建时间 | 日期时间 | 是 | 审计字段 | 系统写入 |
| 18 | `updated_at` | 更新时间 | 日期时间 | 是 | 审计字段 | 系统写入 |
| 19 | `legacy_record_id` | 旧记录ID | 文本 | 否 | 迁移对账用 | — |
| 20 | `legacy_source` | 旧表来源 | 文本 | 否 | 迁移对账用 | — |

### 2.3 飞书字段配置说明

- **`project_id`（自动编号）**：
  - 前缀：`PRJ-`
  - 示例：`PRJ-20260715-001`

- **`customer_link`（关联）**：
  - 关联表：Customer
  - 关联方式：单选（一个项目只关联一个客户）
  - 反向关联：Customer 表可查看关联的项目

- **`project_status`（单选）**：
  - 颜色建议：草稿（灰）、策划中（蓝）、策划已批准（青）、资源确认中（黄）、待拍摄（橙）、拍摄完成（浅绿）、后期制作（绿）、客户确认（紫）、已交付（深绿）、已归档（深灰）
  - **AI 不得直接修改此字段**

- **`planning_link`（关联）**：
  - 关联表：Planning_Document
  - 多选（支持多版本策划案）

### 2.4 视图规范

| 视图名称 | 视图类型 | 筛选条件 | 排序 | 说明 |
|---------|---------|---------|------|------|
| 全部项目 | 表格 | 无 | created_at 降序 | 默认视图 |
| 未来3天拍摄 | 表格 | `project_status` = 待拍摄 且 `shoot_date_start` ≤ 今天+3天 且 `shoot_date_start` ≥ 今天 | shoot_date_start 升序 | F2 自动化数据源 |
| 交付超期 | 表格 | `project_status` ∈ {后期制作, 客户确认} 且 `delivery_due_at` < 今天 | delivery_due_at 升序 | F2 自动化数据源 |
| 待客户确认 | 表格 | `project_status` = 客户确认 | updated_at 降序 | F2 自动化数据源 |
| 策划中 | 表格 | `project_status` = 策划中 且 `planning_status` ∈ {草稿, 待审核, 需修改} | updated_at 降序 | 策划跟进 |
| 资源确认中 | 表格 | `project_status` = 资源确认中 | updated_at 降序 | 资源安排跟进 |
| 高风险项目 | 表格 | `risk_level` = 高风险 | delivery_due_at 升序 | 风险管控 |
| 进行中看板 | 看板 | `project_status` ∉ {草稿, 已归档} | — | 按状态分组 |
| 按负责人分组 | 看板 | 无 | — | project_owner 分组 |
| 近期交付 | 画册 | `project_status` = 已交付 | delivery_due_at 降序 | 已交付项目展示 |

## 3. Resource 统一资源表

### 3.1 目的

合并模特、化妆师、场地、服装、修图师、摄影师等资源的公共信息，以"资源类型 + 视图"替代多张重复子表。通过 `resource_type` 区分资源类别。

### 3.2 字段字典

| # | 字段键 | 显示名 | 飞书字段类型 | 必填 | 说明 | 约束 / 选项 |
|---|--------|--------|-------------|------|------|------------|
| 1 | `resource_id` | 资源编号 | 自动编号 | 是 | 稳定业务 ID | 格式：`RES-{YYYYMMDD}-{###}` |
| 2 | `resource_type` | 资源类型 | 单选 | 是 | 资源类别 | 枚举：模特、化妆师、场地、服装、修图师、摄影师、其他 |
| 3 | `resource_name` | 资源名称 | 文本 | 是 | 名称或称呼 | 非空，≤30 字符 |
| 4 | `contact` | 联系方式 | 文本 | 否 | 按隐私级别控制可见性 | 手机号/微信/邮箱 |
| 5 | `city` | 城市 | 单选 | 否 | 统一城市字典 | 枚举：杭州、上海、北京、深圳、广州、成都、武汉、其他 |

| 13 | `price_max` | 报价上限 | 数字 | 否 | 报价范围上限 | ≥price_min |
| 14 | `rating_quality` | 质量评分 | 评分 | 否 | 质量评价 | 1-5 星 |
| 15 | `rating_reliability` | 可靠性评分 | 评分 | 否 | 合作可靠性 | 1-5 星 |
| 16 | `last_cooperation_at` | 最近合作日期 | 日期 | 否 | 最近一次合作时间 | — |
| 17 | `notes` | 备注 | 多行文本 | 否 | 补充信息，不得代替结构化字段 | ≤500 字符 |
| 18 | `resource_profile_json` | 资源档案JSON | 多行文本 | 否 | 类型特有信息，供 Agent 和 API 使用 | 合法 JSON 字符串 |

| 20 | `updated_at` | 更新时间 | 日期时间 | 是 | 审计字段 | 系统写入 |

### 3.3 飞书字段配置说明

- **`resource_id`（自动编号）**：
  - 前缀：`RES-`
  - 示例：`RES-20260715-001`

- **`resource_type`（单选）**：
  - 颜色建议：模特（粉）、化妆师（紫）、场地（绿）、服装（蓝）、修图师（橙）、摄影师（青）、其他（灰）
  - 此字段是视图分组的核心

- **`resource_profile_json`（多行文本）**：
  - 存储 JSON 字符串，包含类型特有信息
  - 模特示例：`{"height":175, "bust":86, "waist":62, "hip":90, "shoe_size":38}`
  - 场地示例：`{"address":"杭州市西湖区...", "area_sqm":120, "facilities":["自然光","白墙","道具间"]}`
  - 服装示例：`{"category":"婚纱", "sizes":["S","M","L"], "purchase_source":"淘宝", "purchase_url":"..."}`

### 3.4 视图规范

| 视图名称 | 视图类型 | 筛选条件 | 排序 | 说明 |
|---------|---------|---------|------|------|
| 全部资源 | 表格 | 无 | created_at 降序 | 默认视图 |
| 模特库 | 表格 | `resource_type` = 模特 | priority 升序 → rating_quality 降序 | 模特视图 |
| 化妆师库 | 表格 | `resource_type` = 化妆师 | priority 升序 → rating_quality 降序 | 化妆师视图 |
| 场地库 | 表格 | `resource_type` = 场地 | priority 升序 | 场地视图 |
| 服装库 | 表格 | `resource_type` = 服装 | created_at 降序 | 服装视图 |
| 修图师库 | 表格 | `resource_type` = 修图师 | priority 升序 → rating_reliability 降序 | 修图师视图 |
| 摄影师库 | 表格 | `resource_type` = 摄影师 | priority 升序 | 摄影师视图 |
| S级资源 | 表格 | `priority` = S | resource_type 分组 | 顶级资源池 |

#### 11.1.4 转换规则

1. **不允许跨越关键状态直接更新**，除非管理员填写例外原因
   - 关键状态：待定金 → 已成交（需定金确认证据）
   - 关键状态：已成交 → 服务中（需项目进入待拍摄状态）
2. **AI 不得直接修改客户状态**，仅可通过 AI_Inbox 提交候选
3. **S5 自动化只允许单向、受控更新**：
   - 第一个有效项目进入"待拍摄"后，客户可进入"服务中"
   - 所有项目已交付后，客户可进入"已完成"
   - **不允许项目状态直接覆盖"复购/转介绍"或"已流失"**
4. **所有状态变化写入 `Automation_Event`** 或变更日志
5. 公式字段不得作为关键自动化触发源

#### 11.2.4 转换规则

1. **不允许跨越关键状态直接更新**，除非管理员填写例外原因
   - 关键状态：策划已批准 -> 资源确认中（需策划案批准证据）
   - 关键状态：资源确认中 -> 待拍摄（需关键资源已确认）
   - 关键状态：客户确认 -> 已交付（需客户确认证据）
2. **AI 不得直接修改项目状态**
3. **公式字段不得作为关键自动化触发源**
4. **所有状态变化写入 `Automation_Event`** 或变更日志
5. **逆向回退**允许相邻状态回退（如后期制作 -> 拍摄完成），但跨越多步回退需管理员例外原因
6. **S5 自动化联动**：
   - 项目进入"待拍摄"时，客户可进入"服务中"
   - 项目进入"已交付"时，检查客户所有项目是否均已交付
